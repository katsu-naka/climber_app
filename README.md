# アプリケーション名

**Climber（クライマー）**

# アプリケーション概要

- Climberはビジネスユースを想定した進捗管理用タスク管理アプリケーションとなります。  
- タスクを作成して閲覧管理することが出来ます。タスクボードでは、当日のタスクのステータスと進捗率が表示され業務の進行状況が瞬時に把握出来ます。
- 他ユーザーとプロジェクトを作成することでメンバー同士でタスクの共有が行えます。メンバー同士のタスクは自由に閲覧することが可能です。
- 各タスクの詳細ページにはメッセージ機能が備わっており、メンバー同士がコミュニケーションをとることが出来ます。アドバイス、フィードバック、質問などの場としての活用を想定しています。

# Climberが目指す課題解決
### 想定する利用ユーザーについて
- 具体的には下記の人物をメインユーザーとして想定しております。
  - とある企業の営業チームを管理する役職の人物（以下、上司とします。）
  - 上記営業チームの構成メンバーにあたる人物（以下、部下とします。）

### ユーザーが抱える課題
- 上司の課題
  - 部下の当日のタスクを毎朝ミーティングで確認していますが、全員のタスクをヒアリングするのに非常に時間がかかっていました。
  - ミーティングのみでは部下のタスクの内容を全て把握しきれず、後の管理業務でフィードバックが不十分だったり、トラブル対応が後手に回ることがありました。
  - 部下が外出後はタスクに対するフィードバックや質問などのアクションが電話やメールでしかできず、互いのタイミング次第でやり取りに時間がかかってしまいました。
  - 部下が帰社後は当日の活動について報告を依頼するが、報告フォーマットが定まっておらず効率的な管理ができていませんでした。

- 部下の課題
  - 当日のタスクの打ち合わせや、帰社後の報告業務があり営業業務に割ける時間が減っていました。
  - 営業中に遂行中のタスクに問題が発生した場合にメールで上司に報告していたが、タスクについての説明など都度入力しなくてはならず手間だと感じていました。
  - 帰社後の業務報告では本日行った業務をメールやワードに書き起こしていたが、この作業で残業が増えたりと内心煩わしく感じていました。

### Climberで解決できること
- 上司の課題解決
  - 部下には予め当日のタスクの登録を依頼することで、ミーティングでのタスク確認作業を最小限に留めることができます。
  - プロジェクト機能を利用し部下とプロジェクトを作成することで、いつでも部下のタスクを確認できるようになります。
  - メッセージ機能を利用することで、部下が外出後もタスク毎にメッセージを送信し、アドバイスや質問などのコミュニケーションをとることができます。
  - 当日の活動内容がClimberには記録されているので、帰社後の報告業務が不要になります。

- 部下の課題解決
  - 空き時間にタスクを登録できるので営業活動を行う時間を増やすことができます。
  - タスクについて上司への報告や質問があった場合は、該当タスクからメッセージを送れば良いのでやり取りが効率的でスムーズになります。
  - 帰社後の報告業務がなくなるので残業が減ったり、空いた時間を資料作成などに回すことができ、業務の効率が上がります。

# Climberの要件定義

### タスク投稿機能
- タスク一覧表示機能
  - タスクボードには当日のタスクが一覧で表示されます。
  - タスクが登録されていない場合は登録されていない旨のメッセージが表示されます。
  - 一覧表示するタスクの日付はタスクボードのカレンダーから選択して変更することができます。
  - 完了したタスクは「完了」ボタンをクリックすると「完了済」に変更できます。再度クリックすると元に戻ります。
  - タスクボードには当日の「タスクの総数」「完了済タスクの数」「進捗率（％）」が表示されます。
- タスク作成機能
  - 日付、タスク名、説明文（任意）を入力するとタスクが作成できます。
- タスク詳細確認・編集・削除機能
  - 詳細画面では各タスクの詳細が確認できます。
  - 「編集」をクリックするとタスクの内容を編集して更新することができます。
  - 「削除」をクリックするとタスクが削除されます。
### プロジェクト作成機能
- プロジェクト作成、一覧表示、削除機能
  - タスクボードの「プロジェクト選択」をクリックするとプロジェクト一覧画面に移動します。
  - 一覧画面ではユーザーが所属しているプロジェクトが一覧で表示されます。
  - 「プロジェクトを作成する」をクリックするとプロジェクト作成画面へ移動し、プロジェクト名とメンバーを入力すると新規プロジェクトを作成できます。
  - 一覧画面でプロジェクト名をクリックするとプロジェクトの詳細を確認することができます。
- メンバーのタスク閲覧機能
  - プロジェクト詳細画面でメンバーをクリックするとそのメンバーのタスクボードへ移動します。
  - 他メンバーのタスクボードではタスク一覧と詳細及び、タスクステータス、進捗状況を確認することができます。
  - 他メンバー画面ではタスクの作成や編集、削除、ステータス変更などの操作は制限されています。 
### メッセージ機能
- メッセージ送信機能
  - タスク詳細画面よりメッセージを送信することができます。
### ユーザー管理機能
- 新規登録・ログイン（Eメール）
  - ユーザーは名前とEメール、パスワードを入力して新規登録することができます。
  - 登録したEメールとパスワードを使用してアプリにログインすることができます。
- 新規登録・ログイン（SNSアカウント）
  - Googleアカウントを使用してスムーズに新規登録、ログインすることができます。
- ログアウト
  - タスク一覧画面のプルダウンメニューからログアウトをクリックするとログアウトすることができます。
### お問い合わせ送信機能
- お問い合わせ送信
  - タスク一覧画面の「不具合報告、お問い合わせ」をクリックするとお問い合わせフォーム画面へ移動します。
  - 必要事項を入力し送信ボタンを押すと管理者へメールでお問い合わせ内容が送信されます。

# ローカル開発環境での動作手順
- 動作環境について
  - OS macOS Catalina 10.15.7
  - Ruby 2.6.5
  - Ruby on Rails 6.0.3.4
  - yarn 1.22.5
  - git 2.74.3
  - mysql 5.6.50

- 下記のコマンドをターミナルで実行してアプリをクローンしてください。
  - GitHubリポジトリのクローン
  ```
  % git clone https://github.com/katsu-naka/climber_app.git
  ```
  - ~/climber_appへ移動
  ```
  % cd climber_app
  ```
  - Gemのインストール
  ```
  % bundle install
  ```
  - yarnのインストール
  ```
  % yarn install
  ```
  - データベースの作成
  ```
  % rails db:create
  ```
  - テーブル定義の反映
  ```
  % rails db:migrate
  ```
  - アプリの起動
  ```
  % rails s
  ```
